【记_V_vol.1】 二次博客搭建

起因

近来QQ空间写得比较多，然而QQ空间的内容比较随意，如果我想写一些较为严肃的东西，就缺少了一个合适的平台。因此，我就把原先的博客删掉，重新建了一个新的。虽然可能没什么人看，但是这也是我发表文章的一个地方，可以满足我的需求。

然而，这个重建的过程并不顺利。

原理

网上有很多关于使用 hexo+Github Page 创建博客的文章，然而这些文章往往是一上来就说怎么做，几乎没有文章解释原理是什么的（可能是太简单了不配解释吧）。我在这里先凭借我多次失败的经验，把我大概推测出的原理写一下。

首先是在本地通过hexo搭建一个博客，然后再将这个文件夹内的文件都上传到 Github 的一个仓库中，接着访问这个仓库对应的 Github Page ，看到的就是博客了。

这次在“上传”这一操作中遇到了诸多不顺。

困难

我一开始是按照网络上的教程在本地生成了SSH key，然后将该key添加到github账户中，接着直接hexo d推送，多次尝试，都失败了。

最后抱着摆烂的心态，在电脑上安装了Github Desktop，想着通过手动的方式把所有的文件都上传到仓库里面，结果还是不行。

但是，这个时候又试了一次hexo d，突然弹出来一个窗口说要和Github账户绑定，然后经过一系列的验证与确认步骤，竟然成功的推送了。

可能是到了2025年有了新的做法吧，总之最后还是成功了。

特别备注：上传的时候（即执行hexo d时）一定要打开Github Desktop。

其他

在多次访问 Github 的过程中，发现直连的网络质量还是可以的。猜测可能是近期deepseek这样的开源模型利好Github这样的平台。

谢谢大家阅读。
